Array

                        
    
                            
  • Arrays.from()
                                
      
                                    
    • Takes an array-like or iterable object and returns an array.
                                        
        
                                            
      • Strings, Sets, Maps, etc.
        • 
                                        
      
                                    
      • 
                                    
    • Can also be provided with a map function to apply to each element.
      • 
                                
    
                            
    • 
                            
  • for .. of
                                
      
                                    
    • Iterate over an array and return each element.
      • 
                                
    
                            
    • 
                            
  • Array.of()
                                
      
                                    
    • Creates an array from an arbitrary number of arguments.
      • 
                                
    
                            
    • 
                        

                    

                

            

            

                

                    

                        

                            

                        

                        

                            
                            
                        

                    

                    

                        
Array Continued

                        
    
                            
  • Array.fill()
                                
      
                                    
    • Fills all elements of an array with a static value.
      • 
                                    
    • Can also take a start and end index.
      • 
                                
    
                            
    • 
                            
  • Array.find()
                                
      
                                    
    • Returns a value in an array if any element satisfys the testing function.
      • 
                                    
    • Array.findIndex() does the same but returns the index.
      • 
                                
    

                            
    • 
                            
  • Typed Arrays
                                
      
                                    
    • Typed Arrays are also available such as Uint8Array and Float64Array
      • 
                                    
    • Allows you to use native data types for performance benefits.
      • 
                                
    
                            
    • 
                        

                    

                

            

            

                

                    

                        

                            

                        

                        

                            
                            
                        

                    

                    

                        
Collections

                        
    
                            
  • Map
                                
      
                                    
    • Similar to an Object but it has the ability to use non-string keys.
      • 
                                    
    • Optimized to be a performant kv store as opposed to a general Object.
      • 
                                    
    • Can get the size easily with without having to iterate over it like an Object.
      • 
                                
    
                            
    • 
                            
  • Set
                                
      
                                    
    • A unique value store.
      • 
                                    
    • Supports similar operations as map.
      • 
                                    
    • Does not support traditional "Set Theory" operations (union, intersection, etc.) yet.
      • 
                                
    
                            
    • 
                            
  • WeakMap and WeakSet
                                
      
                                    
    • Similar to Map and Set yet key references are "weakly" held.
      • 
                                    
    • Keys are not enumerable or recoverable but performance is much better.
      • 
                                    
    • WeakSets may only hold object values. Useful for fast membership testing.
      • 
                                
    
                            
    •

Generators

                        
    
                            
  • Generators allow you to lazily produce an iterable without having to manage the internal state.
    • 
                            
  • Similar to Python generators, uses the yield keyword.
    • 
                            
  • Declared with function* 
    • 
                            
  • Can use with for .. of or .next()

Arrow Functions

                        
    
                            
  • Anonymous functions with short syntax and a lexically bound "this" value.
    • 
                            
  • Usefull for some functional programming styles where a shorter function is more readable.
    • 
                            
  • Does not re-define "this" and inherits from the outer scope.
                                
      
                                    
    • Avoids that = this pattern.
      • 
                                    
    • Useful for deeply nested functions and some OOP styles.
